### Round 3

时间：

- 2026-04-14

目标：

- 接入 `PydanticAI` 结构化规划，并在无密钥时自动回退
- 补齐 `file_write` / `http_fetch` 执行器与更严格的本地命令策略
- 增强步骤级错误归因与 `agent replay` 可读输出

完成项：

- 定义 `LLMProvider` 协议与 `EnvLLMProvider`
- 新增 `PydanticAIPlanner`（`output_type=LLMStructuredPlan`，`defer_model_check` 便于测试与延迟校验）
- CLI 支持 `UNI_AGENT_PLANNER_BACKEND`：`auto` / `heuristic` / `pydantic_ai`（默认 `auto`：有 `OPENAI_API_KEY` 或 `ANTHROPIC_API_KEY` 时走 LLM，否则启发式）
- 为 `file_write`、`http_fetch` 接入真实 handler（`http_fetch` 支持最大字节截断，URL 仅限 `http/https`）
- `LocalSandbox` 默认命令白名单收紧为 `pwd,ls,cat,echo,rg`，并可通过 `UNI_AGENT_SANDBOX_ALLOWED_COMMANDS` 覆盖
- `PlanStep` 增加 `error_type` / `error_detail`，`Executor` 在失败时写入，`TaskResult.error` 优先使用 `error_detail`
- `agent replay` 增加 `-v/--verbose`，按步骤输出状态与错误归因后再打印完整 JSON
- 新增单元测试与 `tests/integration/test_agent_run.py`（`file_write` 端到端）

当前问题：

- LLM 规划仍依赖外部 API 与网络；结构化计划与任务语义的匹配质量需结合实际任务继续调参
- `HeuristicPlanner` 仍未覆盖写入类自然语言任务，复杂任务需依赖 LLM planner 或显式计划
- `agent replay` 仍为“日志回放 + 步骤摘要”，尚未支持对历史计划自动重执行

验证情况：

- `python3 -m compileall src tests` 已通过
- 在 `.venv` 中执行 `python -m pytest -q` 已通过，结果为 `12 passed`

### Round 4

时间：

- 2026-04-14

目标：

- 提升启发式 Planner 对写入与拉取类任务的覆盖
- 收紧 `http_fetch` 的网络暴露面并统一沙箱超时配置
- 为 LLM Planner 与 CLI 回放补充可配置项与可脚本化输出

完成项：

- `HeuristicPlanner` 支持最小 `file_write` 句式与任务中的 `http(s)` URL 触发 `http_fetch`；当写入路径与读取路径冲突时跳过多余的 `file_read`
- 新增 `http_fetch` URL 安全校验（私网/回环/链路本地/metadata 等，默认拒绝；`UNI_AGENT_HTTP_FETCH_ALLOW_PRIVATE_NETWORKS=true` 可放行）
- `LocalSandbox` 支持 `command_timeout`，与 `UNI_AGENT_SANDBOX_COMMAND_TIMEOUT_SECONDS` 对齐；`http_fetch` 复用同一超时配置
- `PydanticAIPlanner` 支持 `UNI_AGENT_PLANNER_INSTRUCTIONS`、`UNI_AGENT_LLM_TEMPERATURE`、`UNI_AGENT_LLM_RETRIES`
- `agent replay` 支持 `--format steps|jsonl|full`（`jsonl` 末行输出 `task_result` 摘要）
- 增补启发式规划、`http_fetch` 策略、`file_read` 路径穿越与 CLI `jsonl` 回放等测试；集成测试改为真实启发式写入句式

当前问题：

- 启发式写入/抓取仍依赖固定句式，复杂自然语言仍需 LLM Planner
- `http_fetch` 未做 DNS 重绑定防护；仅基于 URL 主机字面量做基线拦截
- Executor 尚未实现失败后的自动重试或局部重规划

验证情况：

- `python3 -m compileall src tests` 已通过
- 在 `.venv` 中执行 `python -m pytest -q` 已通过，结果为 `25 passed`
